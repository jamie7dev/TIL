2. 1주차 git 공부 내용 적어보기

- git 초기화(git initializing): 프로젝트 초반에 생성하고 .git이라는 폴더가 생긴다.
(맥은 shift+command+. 으로 찾을 수 있음)

- commit: 작업 내용 저장. 커밋을 작성함으로써 버전을 관리한다. commit 메시지를 잘 작성해보자.
	커밋 내용은 history로 볼 수 있음.

- add, 스테이징 staging: 커밋에 반영하고 싶은 파일만 선택해서 올린다.
		작업 내역을 잘 묶어서 또는 따로따로 정리해서 올리는 것이 중요하기 때문에 스테이징이 중요함.

- tracking: (협업하기 위해) 원격 repo와 로컬 repo를 연결하는 작업

- pull: 원격 repo -> 로컬 repo / '땅겨온다'라고도 표현
- commit: 로컬 repo에서 커밋 작업 후
- push: 로컬 repo -> 원격 repo / '업로드'라고 생각하면 됨

- clone: 초기 download / 원격 repo에서 (다른 사람 Github) 작업물 전체를 긁어오는 것.
